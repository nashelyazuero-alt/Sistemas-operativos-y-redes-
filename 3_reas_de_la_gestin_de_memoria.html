3 áreas de la gestión de memoria

¿Cuáles son las 3 áreas de la gestión de memoria?

La gestión de memoria opera en tres niveles: hardware, sistema operativo y programa/aplicación. Las capacidades de gestión en cada nivel trabajan conjuntamente para optimizar la disponibilidad y la eficiencia de la memoria.

1. Administración de memoria a nivel de hardware: A nivel de hardware, la administración de memoria se centra en los componentes físicos que almacenan datos , como los chips de memoria de acceso aleatorio ( RAM ) y las memorias caché de la CPU ( L1, L2 y L3 ). La mayor parte de la administración a nivel físico la realiza la unidad de administración de memoria ( MMU ), que controla las operaciones de memoria y caché del procesador . Una de las funciones más importantes de la MMU es traducir las direcciones lógicas utilizadas por los procesos en ejecución a las direcciones físicas en los dispositivos de memoria. La MMU suele estar integrada en el procesador, aunque también puede implementarse como un circuito integrado independiente.

2. Administración de memoria a nivel de sistema operativo: A nivel de sistema operativo, la administración de memoria implica la asignación (y reasignación constante) de bloques de memoria específicos a procesos y programas individuales según las necesidades de recursos de la CPU. Para gestionar este proceso de asignación, el sistema operativo mueve continuamente los procesos entre la memoria y los dispositivos de almacenamiento ( disco duro o SSD ), a la vez que realiza un seguimiento de cada ubicación de memoria y su estado de asignación.

El sistema operativo también determina qué procesos obtendrán recursos de memoria y cuándo se les asignarán dichos recursos. Como parte de esta operación, un sistema operativo puede utilizar el intercambio de memoria (swap ), un método para mover información entre la memoria principal y la secundaria para dar cabida a más procesos.

El sistema operativo también se encarga de gestionar los procesos cuando el ordenador se queda sin memoria física. En ese caso, recurre a la memoria virtual, un tipo de pseudomemoria asignada desde una unidad de almacenamiento configurada para emular la memoria principal del ordenador. Si la demanda de memoria supera la capacidad de la memoria física, el sistema operativo puede asignar automáticamente memoria virtual a un proceso, igual que lo haría con la memoria física. Sin embargo, el uso de memoria virtual puede afectar al rendimiento de las aplicaciones, ya que el almacenamiento secundario es mucho más lento que la memoria principal del ordenador.

3. Administración de memoria a nivel de programa/aplicación: La administración de memoria a este nivel se implementa durante el proceso de desarrollo de la aplicación y la controla la propia aplicación, en lugar de ser administrada centralmente por el sistema operativo o la MMU. Este tipo de administración de memoria garantiza la disponibilidad de memoria suficiente para los objetos y estructuras de datos del programa . Esto se logra mediante lo siguiente:

  • Asignación de memoria: Cuando un programa solicita memoria para un objeto o estructura de datos, esta se asigna a dicho componente de forma manual o automática. Si la asignación es manual, el desarrollador debe programarla explícitamente en el código. Si el proceso es automático, un administrador de memoria se encarga de la asignación, utilizando un componente llamado asignador para asignar la memoria necesaria al objeto. El asignador puede emplear diversas técnicas para asignar bloques de memoria bajo demanda: el método de primer ajuste , en el que asigna el primer bloque libre con el tamaño suficiente para la solicitud; el sistema de bloques contiguos , en el que solo asigna bloques de memoria de ciertos tamaños; o el uso de subasignadores , que asignan pequeños bloques de memoria a una aplicación a partir de un bloque mayor recibido del administrador de memoria del sistema. El administrador de memoria puede estar integrado en el lenguaje de programación o disponible como un módulo independiente.
  • Reciclaje: Cuando un programa ya no necesita el espacio de memoria que se le ha asignado, este se libera y se reasigna (es decir, se recicla). Esta tarea puede realizarla manualmente el programador o automáticamente el administrador de memoria, un proceso que suele denominarse recolección de basura.
